This guide starts a full Higress instance — gateway, controller, and console — on your local machine using the all-in-one Docker image. No Kubernetes required.

## Prerequisites

- [Docker](https://docs.docker.com/get-docker/) installed and running
- Ports 8001, 8080, and 8443 available on your machine

## Start Higress

<Steps>
  <Step title="Create a working directory">
    Higress writes its configuration files to the current directory. Create a dedicated folder first.

    ```bash
    mkdir higress && cd higress
    ```
  </Step>

  <Step title="Start the container">
    ```bash
    docker run -d --rm --name higress-ai -v ${PWD}:/data \
      -p 8001:8001 -p 8080:8080 -p 8443:8443 \
      higress-registry.cn-hangzhou.cr.aliyuncs.com/higress/all-in-one:latest
    ```

    **Port reference:**

    | Port | Purpose |
    |------|---------|
    | `8001` | Higress UI console |
    | `8080` | Gateway HTTP entry point |
    | `8443` | Gateway HTTPS entry point |

    <Note>
      Higress images are hosted on Alibaba Cloud Container Registry, not Docker Hub, so they are not affected by Docker Hub rate limits. Images are scanned by Alibaba Cloud ACR security scanning before publication.
    </Note>

    If the default registry is slow from your location, use a mirror:

    <Tabs>
      <Tab title="China (default)">
        ```bash
        docker run -d --rm --name higress-ai -v ${PWD}:/data \
          -p 8001:8001 -p 8080:8080 -p 8443:8443 \
          higress-registry.cn-hangzhou.cr.aliyuncs.com/higress/all-in-one:latest
        ```
      </Tab>
      <Tab title="North America">
        ```bash
        docker run -d --rm --name higress-ai -v ${PWD}:/data \
          -p 8001:8001 -p 8080:8080 -p 8443:8443 \
          higress-registry.us-west-1.cr.aliyuncs.com/higress/all-in-one:latest
        ```
      </Tab>
      <Tab title="Southeast Asia">
        ```bash
        docker run -d --rm --name higress-ai -v ${PWD}:/data \
          -p 8001:8001 -p 8080:8080 -p 8443:8443 \
          higress-registry.ap-southeast-7.cr.aliyuncs.com/higress/all-in-one:latest
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Verify the container is running">
    ```bash
    docker ps --filter name=higress-ai
    ```

    Wait a few seconds for all processes to start. You should see the container listed with status `Up`.
  </Step>

  <Step title="Open the console">
    Navigate to [http://localhost:8001](http://localhost:8001) in your browser.

    The Higress UI console loads. From here you can manage routes, services, domains, certificates, and plugins.
  </Step>
</Steps>

## Create your first route

The following steps proxy requests sent to `localhost:8080` through to a test backend.

<Steps>
  <Step title="Add a service source">
    In the console, go to **Service Sources** and add a new static address service. For a quick test, you can point it at any reachable HTTP server.

    As an example, use `httpbin.org` on port `80` with scheme `HTTP`.
  </Step>

  <Step title="Create a domain">
    Go to **Domains** and create a new domain entry for `localhost`. Leave the certificate field empty for HTTP-only testing.
  </Step>

  <Step title="Create a route">
    Go to **Routes** and click **New Route**.

    - **Domain**: select `localhost`
    - **Path**: `/test`
    - **Match type**: prefix
    - **Target service**: select the service you created in step 1

    Save the route.
  </Step>

  <Step title="Test the route">
    Send a request through the gateway:

    ```bash
    curl http://localhost:8080/test/get
    ```

    You should receive a JSON response from the upstream service. Route changes in Higress take effect in milliseconds with no reload.
  </Step>
</Steps>

## Stop Higress

```bash
docker stop higress-ai
```

The `--rm` flag in the run command means Docker removes the container automatically on stop. Configuration files persist in your working directory.
